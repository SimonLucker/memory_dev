# Commit Log

This is a plain-language walkthrough of every commit in this repository's history, newest first, grouped by the day the work happened. It exists so a non-developer can understand what changed and why without reading code. The project's arc runs from a visual prototype (a force-directed "memory graph" you could pan, zoom, and filter) to a full proof-of-concept product with a way to record new memories and browse them in a list, to a version tuned and shipped as an iPhone web app, and finally to a Supabase-backed cloud deployment with photo, music, and editing refinements layered on top. Most commits were written by an AI coding agent (Claude) working from a running project brief; two commits were pushed directly by the project's human owner, Simon, and are called out explicitly below.

# 2026-07-19

## d3fac38 — feat(memory): drag thumbnails to reorder a memory's photos
Added drag-and-drop (works with both mouse and touch) so you can reorder a memory's photo thumbnails; the new order is saved and the first photo becomes that memory's thumbnail everywhere it appears. Also cleaned up a few sample memories that had accidentally listed the same photo twice.

## 191d8ef — feat(memorialize): assume 'just now' when no date given; auto-growing composer
If you record a memory without mentioning a date or time, the app now just stamps it with the current moment instead of asking you when it happened. The text box you type into also grows taller as you write (up to about six lines) and shrinks back down after you send.

# 2026-07-18

## 0bd4816 — feat(lightbox): browse all of a memory's photos — swipe on touch, arrows + arrow keys on desktop
The full-screen photo viewer used to show only one image at a time; now it lets you step through every photo attached to a memory with on-screen arrows, keyboard arrow keys, or a swipe on touch screens, including a counter showing your position.

## 45f1069 — feat(music): inline autoplay player in the memory card
The song preview for a memory now plays inline inside the memory card itself (between the photos and the text) and starts automatically when you open that memory, rather than only through the separate floating mini player.

## bfeddd8 — fix(music): server-side catalog lookup with retries; feat: song verification in memorialize
Song lookups used to run in the browser and were unreliable (ad blockers, wrong country's store); they now go through a server-side search with retries and use the phone's own region. While recording a memory, the app now checks the mentioned song right away and shows whether it was found, playable, or needs a correction.

## 7c4a069 — feat(music): Apple Music previews — tap ♪ to play, one tap to the full song
Added the ability to play a 30-second preview of the song tied to a memory (via Apple's public search, no account needed), with a floating mini-player and a button to open the full song in Apple Music.

## 65ca8dc — feat(vault): order-by — time both ways, category, people, feelings, music, places
The memory list ("Vault") can now be sorted and grouped many ways — oldest/newest, category, person, feeling, song, or place — with header labels and counts for each group; memories tied to more than one person or feeling appear under each relevant group.

# 2026-07-15

## c9cf09b — fix(gather): target collision relax, internal edge budget, class-hide no-gather
Refined the "gather" effect that pulls matching memories together when you filter: they now settle without overlapping each other, large groups apply the same connection-thinning rules internally that the rest of the graph uses, and turning off a category in the legend no longer triggers the gathering animation for the remaining items.

## 7d2a2e5 — feat: grouped hover tooltip; person edits create visible links
Redesigned the hover tooltip to group memories by what they share (a person, place, etc.) instead of listing raw connections. Also fixed a bug where tagging a person to a memory didn't reliably create a visible connecting line — every person's memories now always form a connected thread on the graph.

## 811022b — perf: sprite-baked orbs, solid weak threads, sticky throttled labels
A performance pass: memory "orbs" are now pre-drawn once as reusable images instead of being redrawn with expensive glow effects every single frame, fixing stutter when memories first appear. Label text updates were also slowed down and stabilized so labels stopped flickering between different memories every frame.

## faf00f3 — feat(edges): budget allowance decays with graph size
Tuned how many connecting lines are drawn: the limit per memory now shrinks automatically as the total number of memories grows, so a small dataset still looks richly connected while a large one (hundreds of memories) doesn't become an unreadable tangle.

## f29517e — feat(edit): tag editors for people and feelings
Added the ability to edit the list of people and feelings tagged to a memory directly from its card (add or remove as chips), with the graph's connections and legend updating live to reflect the change.

## 42cb249 — feat: memory editing with file persistence; raise edge budget
Added a pencil/edit button to each memory card letting you change its details (what, where, why, summary, importance), with changes saved permanently to the underlying data file via a small local server component. Also allowed more connecting lines to be drawn by default so the graph looks appropriately dense.

## 24ce9db — feat(graph): cluster bridges never fade to invisible
Fixed a visual problem where important connections between two different clusters of memories were so faint they became invisible, making unrelated-looking clusters even though they were actually connected; those "bridge" connections are now guaranteed a minimum visibility.

## 48bd2c1 — fix(layout): anisotropic gravity closes inter-cluster gulfs
Fixed clusters of memories drifting apart into far-flung islands with huge empty gaps between them; added a gentle centering pull (stronger vertically than horizontally, to fit widescreen views) that keeps the whole graph in a coherent, view-fitting shape.

## ebfe1b6 — perf: precomputed layouts; MST edge budget; zoom-constant threads
Fixed noticeable lag when switching between people's memory sets by calculating each person's graph layout ahead of time instead of computing it live on every switch. Also made sure that even with hundreds of memories, the graph only draws the most essential connections by default, and connection lines stay visually consistent (not blobby) as you zoom in.

## 81d7db9 — feat(gather): count-based tightness; ghost off-world edges
